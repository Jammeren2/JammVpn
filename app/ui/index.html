Серверы и узлы

Один локальный прокси: SOCKS5 + HTTP с UDP. Выбран узел — весь трафик идёт через него; «Авто» — по правилам конфига. Наведи на узел для действий: тест задержки, копировать ссылку, экспорт .conf, удалить.

Правила маршрутизации
Сверху вниз, первое совпадение определяет узел. Нет совпадений — узел по умолчанию (выбранный на «Главной»).
Трафик без правил → через узел по умолчанию
Иначе непокрытые правилами соединения идут напрямую. Узел по умолчанию = выбранный на «Главной».
Пресеты:

Правил нет — трафик идёт по настройке «по умолчанию».

Новое правило

Критерии через запятую; между категориями — И, внутри — ИЛИ. Пустое правило = catch-all.
SOCKS5-прокси (мульти)
Несколько локальных SOCKS5: каждый на своём адресе ведёт на свой узел. 0.0.0.0 — открыть в локальную сеть. Один можно сделать системным прокси Windows, остальные указывай вручную в приложениях.

Дополнительно
Локальный WireGuard-сервер
остановлен

JammVPN поднимает свой WG-эндпоинт. Подключаешь устройство по экспортированному .conf — весь его трафик (TCP и UDP) уходит наружу через выбранный узел (основной туннель). Удобно завернуть телефон, VM или отдельное приложение.

QR появится после запуска

Раздельное туннелирование (split)
снято

Приложения задаются правилом с процессом и действием «проксировать» — их трафик (TCP и UDP) перехватывается на уровне NDIS и идёт через выбранный узел; остальные — напрямую. Нужен драйвер WinpkFilter (ndisrd) и запуск от администратора.

Драйвер захвата
WinpkFilter (по умолчанию) или WinDivert (как у WireSock, экспериментально)
Kill-switch
Не выпускать перехваченный трафик при неготовом тоннеле
