# JammVPN — фронтенд для редизайна

Это интерфейс десктопного VPN-клиента (Windows). Технология — **Tauri 2**:
нативное окно + веб-вью, фронт на чистом HTML/CSS/JS (без React/сборки).
Нужно сделать **красивый современный дизайн**, не сломав логику.

## Как посмотреть
Открой **`preview.html`** в любом браузере (Chrome/Edge) — интерфейс
наполнится примерами данных (через `mock-tauri.js`). Окно приложения ≈ **800×620**,
тема тёмная. `index.html` — «боевой» файл (тот же, но без мок-скрипта).

## Файлы
| Файл | Что это | Можно менять? |

| `index.html` | разметка (боевая) | ✅ верстку/классы — да; ❌ `id=...` — нет (см. ниже) |

| `main.js` | логика (вызовы к бэкенду) | ❌ не трогай |

## Что МОЖНО (полная свобода)
- Любые цвета, шрифты, отступы, тени, скругления, иконки, анимации.
- Переверстать `index.html`: контейнеры, сетки, порядок и группировку блоков,
  сворачиваемые секции, вкладки/сайдбар вместо длинной простыни — как угодно.
- Менять `style.css` целиком. Тема задаётся переменными в `:root` (`--bg`,
  `--panel`, `--accent`, `--ok`, `--err` и т.д.) — удобно для перекраски.
- Добавлять свои элементы, картинки, SVG-иконки.

## Что НЕЛЬЗЯ ломать (иначе перестанет работать)
`main.js` находит элементы по `id` и по нескольким классам-«крючкам». Их **нельзя
переименовывать или удалять** (перемещать/перекрашивать/оборачивать — можно).

1. **Все атрибуты `id="..."`** в `index.html` — сохрани как есть (кнопки, поля
   ввода, select'ы, контейнеры-`<tbody>` для списков, элементы-индикаторы).
2. **Пустые контейнеры списков** (`<tbody id="nodes-body">`, `id="subs-body">`,
   `id="rules-body">`) — в них JS вставляет строки. Структуру строки (колонки)
   можно стилизовать, но строки генерит скрипт — не вставляй контент руками.
3. **Классы-крючки** (JS вешает на них поведение и состояние — оставь имена):
   - `status` + модификаторы `on`/`off` (бейдж статуса прокси);
   - `split-state` + `on`/`off` (статус split);
   - `hint` + `ok`/`err` (строки-сообщения под формами);
   - `lat` (+ `ok`/`err`), `geo-ind` (+ `ok`/`err`) — индикаторы;
   - `button.x` (удалить), `button.mini` (▲▼✎ в правилах) — генерятся скриптом;
   - в правилах строки несут `data-i`, `data-act`; в узлах — `data-name`;
     в подписках — `data-url`. Эти `data-*` нужны логике — не убирай.
4. `<datalist id="node-names">` — служебный, оставь.

> Правило простое: **меняй вид, сохраняй `id` и перечисленные классы/`data-*`.**
> Если хочешь убрать блок из вида — лучше спрячь через CSS, чем удаляй из HTML.

## Структура экранов (блоки `.panel`)
1. **Шапка** — логотип/название + бейдж статуса прокси.
2. **Локальный прокси (SOCKS5)** — адрес, выбор узла, Запустить/Остановить.
3. **Узлы** — таблица серверов, тест задержек, удаление.
4. **Импорт** — поле ссылки/подписки + кнопки.
5. **Подписки** — список + форма добавления.
6. **Geo-базы** — пути к geosite/geoip + индикаторы.
7. **Настройки маршрутизации** — режим по умолчанию + узел по умолчанию.
8. **Система** — автозапуск.
9. **Раздельное туннелирование (split)** — режим, приложения, kill-switch и т.д.
10. **Правила маршрутизации** — таблица правил + форма (это и №9 — продвинутые,
    можно вынести на отдельную вкладку/«Дополнительно»).

Идеи приветствуются: сайдбар-навигация по разделам, карточки узлов вместо
таблицы, онбординг при пустом списке, тёмная/светлая тема через те же переменные.
